**目标**：把 HermesGo 收敛成一个免费的、可复制的、可直接使用的 Windows 启动包，避免付费 launcher、避免多窗口重复弹出，保留可见监督和自动重试。

## 1. 免费化原则

1.1 只保留公开可获取、免费可用的本地组件。

1.2 运行路径以 Docker + Ollama + Hermes Agent 为主，不依赖 Pinokio。

1.3 若本机缺少 Docker，只打开免费的官方说明，不引入付费服务。

1.4 目录内不再保留 tar 导出包作为主功能。

## 2. 单窗口监督

2.1 用户双击 `HermesGo.bat` 后，默认进入监督器。

2.2 监督器负责拉起 headless 主程序，监控日志，识别成功、失败和超时。

2.3 监督器的子进程窗口要隐藏，避免重复弹出多个相同窗口。

2.4 启动失败时，要在监督窗口里直接可见，并写入日志文件。

## 3. 自动重试

3.1 默认启用自动重试。

3.2 每次失败后按延迟重试，不在第一轮失败时直接放弃。

3.3 若多次重试仍失败，保留错误状态，等待人工介入。

## 4. 当前实现

4.1 `HermesGo.config.bat` 去掉了 Pinokio 路径，改为 Docker 免费文档链接。

4.2 `HermesGo.bat` 只保留 Docker 主路径和免费文档回退。

4.3 `HermesGoSupervisor.ps1` 启动子进程时使用隐藏窗口，避免重复窗口。

4.4 `HermesGoSupervisor.bat` 默认带上自动重试参数。

4.5 `README.md` 已改写为免费版说明。

## 5. 验证要求

5.1 继续保留 headless 自检，保证主流程不坏。

5.2 监督器要单独验证成功、失败和重试路径。

5.3 若后续再出现重复窗口，优先检查子进程启动方式，而不是再加新的入口窗口。
