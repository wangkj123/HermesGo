## 1. 这次新增了什么

1.1 `HermesGo.config.bat`

1.1.1 作为 HermesGo 的共享配置入口，统一保存镜像名、模型名、端口和常用 URL。

1.2 `Verify-HermesGo.ps1`

1.2.1 在无真实 Docker 的机器上模拟完整流程。

1.2.2 验证缓存镜像分支、联网拉取分支和导出分支。

1.3 `Verify-HermesGo.bat`

1.3.1 提供单文件双击入口，方便在 Windows 上直接执行验证。

## 2. 当前规则

2.1 启动器和导出器都从 `HermesGo.config.bat` 读取统一配置。

2.2 验证脚本会临时创建 `docker.cmd` 桩，不依赖真实 Docker 也能把关键路径跑通。

2.3 验证脚本默认清理临时目录，避免把模拟产物留在仓库里。

## 3. 已验证路径

3.1 缓存 tar 存在时，启动器会先 `docker load` 再 compose 启动。

3.2 缓存 tar 不存在时，启动器会回退到 `docker pull`。

3.3 导出脚本会执行 `docker pull` 和 `docker save`，把镜像写入 `images/`。

## 4. 价值

4.1 规则化的共享配置，减少了后续修改镜像名时的遗漏风险。

4.2 自检脚本把“看上去能跑”变成“可以复现地证明能跑”。

4.3 这更接近你要的绿色包目标：目录可搬运、流程可验证、问题可回放。
