**目的**：把 HermesGo 的“绿色版”再往前推进一层，补上 Docker 镜像缓存、导出和导入流程，让目录更接近“拷贝即用”。

**状态**：已被 `docs/015` 取代，保留作历史记录。

## 1. 为什么要补这一层

1.1 仅有 `docker-compose.yml` 只能说明怎么跑，不能保证搬到别的机器上后还马上有镜像。

1.2 绿色版真正有价值的部分，不只是配置和日志，还包括 Docker 镜像本身。

1.3 所以 HermesGo 现在增加镜像缓存目录 `images/`，并增加镜像导出脚本 `Export-HermesGoImages.bat`。

## 2. 当前约定

2.1 `HermesGo/images/ollama_ollama_latest.tar`

2.1.1 存放 `ollama/ollama:latest` 的镜像归档。

2.2 `HermesGo/images/hermes_agent_latest.tar`

2.2.1 存放 `ghcr.io/nousresearch/hermes-agent:latest` 的镜像归档。

2.3 `HermesGo.bat`

2.3.1 启动时先检查本地镜像是否存在。

2.3.2 如果本地缺镜像但目录里有 tar，就先 `docker load -i`。

2.3.3 如果 tar 也没有，就回退到 `docker pull`。

## 3. 导出流程

3.1 在一台已经装好 Docker 的机器上运行 `Export-HermesGoImages.bat`。

3.2 该脚本会先拉取镜像，再用 `docker save` 导出到 `HermesGo/images/`。

3.3 导出完成后，把整个 `HermesGo/` 目录复制到目标机器。

## 4. 导入流程

4.1 目标机器启动 `HermesGo.bat`。

4.2 如果 Docker 已安装，但本机还没有镜像，启动器会先从 `images/` 加载。

4.3 如果目标机器也没有 tar 文件，启动器再执行网络拉取。

## 5. 验证状态

5.1 已补齐 `images/README.md`。

5.2 已补齐 `Export-HermesGoImages.bat`。

5.3 已把 `HermesGo.bat` 改成优先加载 tar，再 pull 的路径。

5.4 当前机器没有真实 Docker，因此没有执行真实 `save/load`，但脚本逻辑已经落地并通过假 Docker 分支验证。

## 6. 下一步

6.1 如果后面你要把这个包真正搬到别的机器，先执行一次镜像导出，再整体复制目录。

6.2 如果你想再压缩一步，可以继续把镜像版本从 `latest` 固定成某个发布 tag。
