# 进度日志与归档执行规则

**文档编号**：021

**生成日期**：2026-04-16

**最后修改**：2026-04-16

**参考文档**：
- docs/001-2026-04-16-0825-AI更好编程与可理解性总纲.md

**目的**：把“做事有记录”落成仓库内的可执行规则，区分实时进度日志与阶段性归档文档，避免过程只留在对话里。

## 1. 进度日志

1.1 任务执行时持续记录进度日志。

1.2 每条记录至少包含：
- 时间
- 动作
- 使用工具
- 结果
- 下一步

1.3 进度日志默认落盘到仓库根目录 `logs/agent-progress.md`。

1.4 进度日志的作用是保留过程证据，便于追查“做了什么、怎么做的、做到哪一步了”。

## 2. 归档

2.1 当任务形成稳定阶段性结果时，再整理成 `docs/` 下的正式文档。

2.2 归档文档用于沉淀方案、结论、验证和风险，不能用进度日志替代。

2.3 归档继续沿用现有编号体系，保持文档顺序可追踪。

## 3. 执行方式

3.1 对话里继续做即时进度说明。

3.2 仓库内同步写入 `logs/agent-progress.md`。

3.3 到达稳定阶段后，按规则输出 `docs/` 归档文档。
