**目的**：把 HermesGo 从“带镜像缓存的便携包”收敛成“直接使用包”。这个版本不再默认保留 tar，不再把镜像导出当主体功能，而是以现成 Docker 环境为前提，启动后主动做 hello 探测。

**状态**：已叠加 `docs/017` 的可见状态窗口方案。

## 1. 当前定位

1.1 HermesGo 是一个使用包，不是安装包。

1.2 默认目录只保留启动、配置、自检和日志相关内容。

1.3 不再以 tar 镜像缓存作为默认交付物。

## 2. 启动流程

2.1 双击 `HermesGo.bat`

2.2 脚本优先检测 Docker Compose

2.3 启动 `ollama`

2.4 拉取 `hermes3`

2.5 启动 `hermes`

2.6 自动对 `http://localhost:3000` 做 hello 探测

2.7 探测通过后，才把这个目录视为“直接可用”

## 3. hello 探测

3.1 hello 探测的目的，是确认不是“命令跑了就算成功”，而是真正拿到本地服务响应。

3.2 探测优先使用 `curl`

3.3 如果没有 `curl`，回退到 PowerShell 的 `Invoke-WebRequest`

3.4 只要服务有响应，就记录为成功，不再要求用户手工再点一遍

## 4. 验证方式

4.1 `Verify-HermesGo.bat` 会模拟 Docker 和 curl 的返回。

4.2 验证重点是：

4.2.1 启动脚本确实会走 Docker 分支

4.2.2 启动脚本确实会做 hello 探测

4.2.3 失败时会停在可读的日志里，而不是沉默退出

## 5. 目录收敛

5.1 删除镜像导出脚本

5.2 删除 tar 默认缓存说明

5.3 删除镜像搬运作为主线的叙述

5.4 只保留直接使用所需文件

## 6. 结论

6.1 这个版本才符合你说的“不是安装包，是使用的包”。

6.2 tar 可以存在于单独的离线扩展方案里，但不应出现在默认交付包里。

6.3 当前包的判断标准是：能启动、能探测、能回应。
