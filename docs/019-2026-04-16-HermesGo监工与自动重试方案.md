**目的**：把 HermesGo 的“看着我干活”的要求落成一个真正的监工层。监工窗口负责拉起 headless 的 HermesGo 运行，盯住日志和退出结果，出错时高亮并可自动重试。

## 1. 默认入口

1.1 用户双击 `HermesGo.bat`

1.2 `HermesGo.bat` 不再先弹旧状态窗，而是直接打开 `HermesGoSupervisor.bat`

1.3 监工窗口再去拉起 headless 的 `HermesGo.bat`

## 2. 监工职责

2.1 监工窗口负责显示启动进度、日志尾部和最终结果。

2.2 监工会识别 `HermesGo finished with exit code X.` 这样的收尾行。

2.3 如果退出码是 0，监工标绿并保持完成状态。

2.4 如果退出码不是 0，或在超时时间内没有完成，监工标红并发出失败提示。

## 3. 自动重试

3.1 默认开启自动重试。

3.2 出错后，监工会按延迟重试，最多重试若干次。

3.3 如果重试仍失败，监工保留错误状态，等待人工重启。

## 4. 监工与状态窗口的关系

4.1 旧的 `HermesGoStatus.ps1` 仍可作为日志查看器保留。

4.2 但默认交互入口已经切换成 `HermesGoSupervisor.ps1`。

4.3 以后用户看到的应该是“监工窗口”，不是“单纯日志窗”。

## 5. 验证方向

5.1 继续保留 headless 验证，确认主流程不坏。

5.2 后续再补一个针对 supervisor 的专门验证脚本，检查启动、失败和自动重试路径。

5.3 当前阶段已经满足“错误要被看见、失败要被重试”的最低要求。
