**目的**：把 HermesGo 的“看得见”补齐，让启动时有一个明确的可见状态窗口，而不是只依赖命令行输出。

## 1. 为什么要加这个窗口

1.1 用户双击后，需要能直接看到启动进度。

1.2 仅有命令行输出，容易在 IDE、隐藏终端或后台环境里看不到。

1.3 这个窗口专门用于显示日志和状态，不承担安装职责。

## 2. 当前实现

2.1 `HermesGoStatus.ps1`

2.1.1 使用 WinForms 打开一个独立窗口。

2.1.2 实时读取 `HermesGo/logs/HermesGo.log`。

2.1.3 提供复制日志和打开日志文件按钮。

2.2 `HermesGo.bat`

2.2.1 默认启动时会拉起状态窗口。

2.2.2 如果设置 `HERMESGO_HEADLESS=1`，则不打开状态窗口，方便自检。

## 3. 可见性目标

3.1 用户能看到窗口正在启动。

3.2 用户能看到 `Docker Compose detected`、`Hermes container launched`、`Hello probe succeeded` 这些关键状态。

3.3 如果后续失败，窗口里会留下最后一条可读日志，而不是黑箱退出。

## 4. 自检原则

4.1 自检默认 headless，不额外弹窗口。

4.2 自检仍然要验证正常模式下的日志生成逻辑。

4.3 可见窗口是给用户看的，不是给验证脚本看的。
