# Workflow

本目录保存项目级自动化开发流程。

- `roles.md`：团队角色和职责边界
- `stages.md`：阶段流程和门禁
- `Start-AutoDevFlow.bat`：可双击入口，运行后自动打开最新结果说明和运行目录
- `Start-AutoDevFlow.ps1`：启动一次开发流程，生成运行目录、过程文档、验证结果和审查记录
- `latest-summary.txt`：最近一次运行的结果说明
- `latest-run.txt`：最近一次运行目录路径
- `runs/`：每次流程运行的产物

这套流程面向“像实际开发团队一样工作”的场景，强调：

- 先目标和契约，再实现
- 实现、监工、测试、审查分角色
- 过程、结果和审查都落盘
